## -1. 签证/免签状态验证（硬规则，违反即整条作废）

**核心铁律：禁止凭训练数据推断任何国家对中国公民的签证政策。签证/免签状态必须从下方「已验证签证表」查表获取；表中未覆盖的国家必须走 WebSearch 验证后再写入清单。**

**为什么是硬规则**：签证政策按国籍区分，AI 极易混淆"对日本/韩国/新加坡护照免签"与"对中国护照免签"。错误的"免签"标注会导致用户不办签证直接出发、在值机柜台被拒绝登机，后果远比漏带一件衣物严重。

**执行流程**：
1. 目的地确定后，先查下方「已验证签证表」
2. 命中 → 直接使用表中描述，写入证件清单的签证条目 note
3. 未命中 → WebSearch 搜索「{国家英文名} visa requirements for Chinese passport holders {当前年份}」，从官方来源提取签证类型+停留天数+办理方式，写入 note 并标注"🔍 搜索结果"
4. **绝不允许**在 label 或 note 中写"免签 X 天"除非有表中明确记录或 WebSearch 官方来源确认

## 0. 分类归属唯一性（硬规则，违反即废稿重生成）

**核心铁律：每个物品只能出现在唯一一个 category-card 中，禁止跨类重复。**

生成清单时必须逐项自检：

1. **一物一卡**：同一件物品（或同一用途的物品）只能归入一个 category-card。例：
   - 防晒霜 → 只在「洗漱与个护」，不得同时出现在「衣物」「特殊装备」
   - 充电宝 → 只在「电子设备」，不得同时出现在「交通方式专属」
   - 泳衣 → 只在「特殊装备·潜水/水上」或「休闲度假·海边」（取其一），不得同时出现在「衣物」
   - 护照/签证 → 只在「证件与文件」，不得同时出现在「国际出行专属」

2. **分类边界映射表**（只列易错项；未列出的物品走常识判断即可）：

   | 常错物品 | 唯一归属 |
   |---------|---------|
   | 防晒霜 / 防蚊液 / 润唇霜 / 晒后修复 | 🧴 洗漱与个护（不进药品或特殊装备） |
   | 无人机 / 转换插头 / 充电宝 / 充电器 / 耳机 | 📱 电子设备（不进摄影 / 交通） |
   | 相机 / 镜头 / 三脚架 / 存储卡 | 📷 摄影器材（不进电子设备） |
   | 车载充电器 / 手机支架 / 应急三角牌 | 🚗 交通方式专属·自驾（不进电子设备） |
   | 护照 / 签证 / 机票行程单 / 保险单 / 邀请函 | 📄 证件与文件（不进国际出行专属） |
   | 泳衣 / 沙滩巾 / 浮潜面镜 | 🏖️ 特殊装备·水上 或 休闲度假·海边（不进衣物） |
   | 滑雪镜 / 滑雪手套 / 护膝 / 潜水袜 / 涉水鞋 / 雪套 / 防沙头巾 | 对应活动的特殊装备卡（不进衣物） |
   | 儿童防晒 / 水壶 / 玩具 / 绘本 | 👶 亲子装备（不进洗漱 / 摄影 / 衣物） |
   | 头灯 / 手电 / 户外炊具 / 帐篷 / 睡袋 | 户外/露营特殊装备（不进电子设备） |
   | 颈枕 / 眼罩 / 耳塞 / 压缩袜 | ✈️ 交通方式专属（不进衣物） |

   **未覆盖场景**：用户提到冷门活动 / 交通 / 目的地时，走 `WebSearch「{活动或国家英文名} packing list essentials {year}」` 取前 3 条可靠结果的物品，AI 常识判断归属；WebSearch 无可靠结果时用类比推理并在清单头部标注"⚠ AI 推荐，建议出行前核实"。

3. **重复检测规则**：生成完 body HTML 后，扫描所有 `<label>` 文本：
   - 同一物品名（忽略数量后缀 ×N）不得出现在两个不同的 category-card 中
   - **同一 category-card 内同一物品也不得出现多次**——即使 note 不同也算重复（如"短袖T恤 ×3"和"短袖T恤/速干衫 ×2"算重复，应合并为一条）
   - 若发现跨卡片重复，保留"边界映射表"指定的那个分类，删除另一处
   - 若发现卡片内重复，合并为一条（数量取较大值，note 合并关键信息）

4. **"国际出行专属"分类的特别约束**：
   - 此分类**只放**无法归入其他任何分类的国际特有实物（如液体分装袋、防水袋）
   - 证件类 → 归「证件与文件」；电子类 → 归「电子设备」；衣物类 → 归「衣物」
   - 此分类不是"国际行程所有额外物品的垃圾桶"

5. **特殊装备分类的约束**：
   - 按活动类型命名子卡片（如"📷 摄影器材""🤿 潜水装备"）
   - 子卡片内只放该活动的**专用装备**，通用物品（衣物、洗漱、电子）归各自分类
   - 例：摄影器材卡片内只放相机/镜头/三脚架等，绝不放防晒霜、儿童用品、衣物

## 1. 基础分类结构

1. **证件与文件**
   - 国内出行：身份证、机票/车票行程单、酒店预订确认单
   - 国际出行（以下条目全部归入本分类，**不得**另建"出入境文件"独立分类）：
     - 护照（有效期 > 6个月）、签证/签注 + 打印件、身份证（备用）
     - 往返机票行程单（打印+电子版，入境时出示）
     - 酒店预订确认单（打印件，入境可能查验）
     - 旅行保险单（打印件，含对应活动类型）
     - 证件照2张、护照复印件（与原件分开存放）、电子扫描件存手机/邮箱
     - 商务邀请函/会议确认函（商务出行时，入境时出示）
     - 笔（填写入境卡）
     - 无人机登记证明（携带无人机时，如适用）

2. **财务**
   - 国内（默认仅列必带项 + 可选应急项）：
     - 必带：手机支付（微信/支付宝）
     - 可选（应急备用，标 note "手机丢失/没电时备用"）：少量现金、银行卡
     - 自驾追加：ETC 通行卡
     - ⚠ 不再默认列出银行卡 / 信用卡 / 现金作为必带项，2026 年国内出行手机支付已覆盖城市游/景区游 99% 场景
   - 国际：目的地货币现金、支持境外交易的信用卡（Visa/Mastercard）、零钱包/硬币包

3. **衣物**（基于天气、天数、交通方式计算）
   - 数量计算逻辑（**公式：ceil(天数 / 频率)**，向上取整）：
     - 内衣：每2天1件 → ceil(天数/2)。例：3天→2件，5天→3件，7天→4件
     - 内裤：每2天1件 → ceil(天数/2)。例：3天→2件，5天→3件，7天→4件
     - 袜子：每2天1双 → ceil(天数/2)。例：3天→2双，5天→3双，7天→4双
     - 上衣（T恤/衬衫）：每2天1件 → ceil(天数/2)。例：3天→2件，5天→3件，7天→4件
     - 裤子/短裤：每3天1条 → ceil(天数/3)。例：3天→1条，5天→2条，7天→3条
     - 外套：按温度段准备1-2件（不随天数增加）
     - 睡衣：1套（行程 > 10天可带2套）
   - **数量与备注必须自洽**：item label 中的数量（如 ×N）必须与 note 中的频率描述逻辑一致。若 note 写"每2天1件"，则 N = ceil(天数/2)，N 不得等于天数
   - 温度决定衣物类型（见下方温度-衣物映射）
   - 出行类型决定特殊衣物（商务正装/户外速干/滑雪服等）
   - 温度-衣物映射：
     - ≤ 0°C：羽绒服、保暖内衣、毛衣、围巾、手套、帽子、厚袜子
     - 1-10°C：厚外套、毛衣/卫衣、长裤、薄围巾
     - 11-20°C：薄外套/夹克、长袖T恤、长裤
     - 21-28°C：短袖T恤、短裤/裙子、薄长裤（空调房）
     - ≥ 29°C：短袖/背心、短裤/裙子、防晒衣、遮阳帽、墨镜

   **衣物按人拆卡渲染规则**（仅当 SKILL.md Step 4.2 判定为"拆卡"时生效）：拆成 N 个「👕 衣物 - {label}」子卡片；件数独立按 ceil(天数/频率) 计算；性别未知走中性品类（裤子路线）；儿童 0-3 岁统一「换洗衣物 ×N 套」+ 口水巾，4-12 岁频率+1（每天 1 件上衣）；老人追加 1 件保暖内衣；共享物品（防晒衣/遮阳帽/雨伞）只在"我"卡出现并 note "全家共用"。不拆卡时单卡「👕 衣物」按用户性别或中性出。

4. **洗漱与个护**
   - 基础：牙刷、牙膏、毛巾（或一次性压缩毛巾）、洗面奶、梳子
   - 防晒：SPF50+防晒霜、晒后修复（温度 > 25°C 或海边目的地）
   - 女性专属：生理期用品（卫生巾/棉条/月经杯）、女性洗液
   - 化妆包（按需）

5. **电子设备**
   - 手机 + 充电器 + 数据线
   - 充电宝（飞机限带 ≤ 20000mAh，需有3C认证标识）
   - 耳机
   - 国际出行：转换插头（按目的地推荐具体型号）
     - 日本/美国/加拿大：A型（两脚扁插）
     - 英国/香港/新加坡/马来西亚/阿联酋：G型（三脚方插）
     - 欧洲大部分（法国/德国/意大利/西班牙/荷兰/瑞士/俄罗斯/土耳其）：C/F型（两脚圆插）
     - 澳大利亚/新西兰：I型（三脚斜插）
     - 韩国：C/F型（两脚圆插）
     - 泰国/越南/柬埔寨/印尼/菲律宾/老挝：A/C/O型混合（建议带万能转换头）
     - 文莱：G型（三脚方插，与英国相同）
     - 印度/尼泊尔/斯里兰卡：C/D/M型（三脚大圆插，6A/15A两种规格）
     - 南非：M型（三脚大圆插）
     - 以色列：H型（三脚扁插）
     - 巴西：C型 + N型（NBR 14136 标准，三脚斜圆插，需专用转换头）
     - 阿根廷：I型（三脚扁插，与澳大利亚相同）
     - 智利：C型 + L型（意标三脚扁插）
     - 秘鲁：A型 + C型（两脚扁插 + 两脚圆插）
     - 墨西哥/巴拿马/哥伦比亚：A型（两脚扁插，与美国相同）
   - 国际出行：国际漫游SIM卡或eSIM（按目的地推荐）

6. **药品与健康**（按目的地 + 季节动态组合）
   - 基础常备（所有行程）：感冒药、退烧药、创可贴 ×5、体温计、云南白药气雾剂
   - 交通场景：晕车药（含儿童剂型）、晕船药（邮轮/海岛）
   - 肠胃场景：止泻药、蒙脱石散、益生菌、口服补液盐（东南亚/印度/中东优先）
   - 过敏场景：氯雷他定/西替利嗪、鼻腔喷雾（花粉季日韩 3-5 月 / 华北沙尘暴 / 中亚沙尘）
   - 热带 / 夏季场景（气温 ≥ 30°C 或热带目的地）：藿香正气水、防中暑十滴水、电解质补充剂、驱蚊液（防登革热/疟疾）、清凉油
   - 寒冷 / 冬季场景（气温 ≤ 5°C）：润喉糖、护手霜、暖宝宝、感冒清热颗粒
   - 高原场景（海拔 > 2500m，触发即加）：红景天（提前 1 周服用）、高原安、便携氧气瓶
   - 国际出行追加：处方药英文说明（部分药品海关须凭说明入境）、常用药中英对照单
   - 特殊需求：个人处方药（带够全程用量 + 3 天余量；老人加至 5 天余量）
   - 口罩/鼻腔喷雾：花粉季、雾霾季、极寒地区必备
   - ⚠ 防蚊液归「洗漱与个护」（见§0映射表），本分类不列；防蚊场景在此处仅提示"见洗漱个护卡"

7. **国际出行专属**（仅国际出行时显示）

   ⚠ **强制边界规则（防止"提醒"被错误渲染为可勾选清单）**：
   本节内容必须按"渲染目标"二分，**禁止**混进同一个 category-card，**禁止**新建名为"出入境/海关/申报/出行提醒/出入境文件"的 category-card 把告知项或证件文件渲染成 checkbox：

   - **实物 / 文件 / 行前可勾选动作 → 进对应分类清单**（如「证件与文件」「财务」），可勾选
     - 例：EVUS 登记打印件、I-94 入境记录截图、紧急联系卡纸质版、入境卡填写笔
     - ⚠ **证件/文件类条目（护照、签证、机票行程单、酒店确认单、保险单、邀请函等）必须合并进第1节「证件与文件」分类，不得另建独立的"出入境文件"分类卡片**
   - **告知 / 警示 / 法规 / 紧急联系电话 → 全部进 info-card「紧急联系与出行提醒」**，不可勾选
     - 例：禁带肉类/水果/土壤、现金 > $X 须申报、大使馆电话、中药材谨慎、签证时效

   **判定自检**：任一拟入清单的 item label 命中下列任一条件，必须迁出到 info-card：
   - 句首含"⚠"、"禁"、"勿"、"须申报"、"注意"、"提醒"、"警告"
   - 内容是法规条款 / 数字阈值 / 紧急号码 / 联系电话 / 网址 / 时效要求
   - 用户无法通过"打包动作"勾选完成（如"EVUS 状态最新"是状态确认，不是物品）

   下列条目**只用于驱动 info-card 内容生成**，**不**渲染为分类卡片：
   - 大使馆/领事馆紧急联系电话
   - 目的地紧急电话（报警/急救/火警）
   - 入境所需文件清单（按国家）
   - 海关申报注意事项（按国家/地区）：

     **亚洲：**
     - 日本：中国公民需提前办理旅游签证（单次/三年/五年）；需填写入境卡 + 海关申报单，禁止携带肉类、水果、种子等动植物制品
     - 韩国：中国公民需提前办理签证（济州岛免签30天除外）；禁止携带肉类、水果、种子，畜产品需检疫证明
     - 泰国：禁止携带肉类、新鲜水果，现金 < 20,000泰铢无需申报
     - 新加坡：禁止携带口香糖（除药用）、肉类、新鲜水果，电子烟禁止入境
     - 马来西亚：免签30天（每180天累计不超90天），需填电子入境卡；东马入境需酒店订单、返程票及1000美元现金；酒类限1升，食品不超150马币；严禁毒品
     - 越南：普通护照需办签证（电子签或使馆签），入境需填写出入境申报表，禁止携带肉类、新鲜水果、植物种子

    - 反推装备的 `note` 与普通 note 一致样式，仅文案前缀带 Day 编号
    - 若清单中反推装备 ≥ 5 件，可在分类卡片底部加一条小字"基于行程文件的活动建议"（不强制）

## 2. 紧急联系/提醒区块（info-card）内容生成

每个行程都生成此区块，位于每日穿搭建议之后、操作栏之前。

**基础内容**（所有行程必出）：
- 报警电话、急救电话、火警电话（国内：110/120/119；国际：按目的地国家填写）
- 目的地最近医院名称 + 电话（按目的地城市查询）

**特殊提醒内容**（按行程条件动态追加）：
- 高原目的地（海拔 > 2000m）：高反注意事项、最近高压氧舱位置
- 户外徒步行程：山区救援电话、最近救援站位置
- 国际出行：大使馆/领事馆电话、当地紧急电话
- 亲子游：儿童走失应对提示、目的地儿童医院电话
- 雨季目的地：地质灾害预警电话、避险提示

**标题动态规则**（根据实际内容组合自动选择）：
- 仅基础紧急电话 + 医院地址 → 标题「📞 紧急联系信息」
- 仅有特殊提醒（无紧急电话） → 标题「️ 出行提醒」
- 基础紧急电话 + 特殊提醒同时存在 → 标题「 紧急联系与出行提醒」

## 3. 按天穿搭建议生成逻辑（必出）

按天穿搭建议为必出模块，必须生成。逻辑如下：

1. **数据准备**
   - 获取每个目的地城市每日的天气预报（最高温、最低温、天气代码）
   - 多城市行程时，按城市分段，每个城市单独计算

2. **穿搭推荐规则**
   - 基于当日温度范围和天气状况，从温度-衣物映射表中匹配推荐衣物
   - 考虑出行类型的特殊衣物需求（如商务正装、户外速干等）
   - 考虑特殊地理/气候场景（如高原需加强保暖、海边需防晒等）

3. **输出格式**
   - 每天一行，格式：`Day N (M/D): 最低-最高°C 天气描述`
   - 下方列出推荐穿搭：`建议：外套 + 上衣 + 裤子 + 配件`
   - 多城市行程时，在城市切换处插入城市标题分隔

4. **特殊情况处理**
   - 如果天气 API 失败使用气候平均值，穿搭建议仍正常生成，但标注"基于历史平均温度，建议出行前再次确认"
   - 如果行程超过 14 天，仍逐天生成，不省略

5. **跨城日提醒（多城市行程必出）**

   凡是同一天涉及城市切换的 Day（例：Day3 上午在洛杉矶、下午飞拉斯维加斯），当日穿搭段落末尾追加固定提示句：
   - 模板："🚌 换城日：{上午城市} → {下午城市}，出发前把厚外套 / 备用衣物塞入行李箱，随身包只留证件 + 手机 + 充电宝 + 一件保暖单品"
   - 生成条件：该日在"多城市天数划分"结果中处于城市 A 的最后一天且下一天为城市 B，或用户在文件解析中标注了跨城交通节点

6. **长清单去重扫描（收尾必做）**

   - 生成完所有 category-card 后，扫描一次全局 label token（去掉 ×N 数量后缀、去掉括号内附注）
   - 同名 token 出现在 ≥ 2 张卡片 → 保留 §0 映射表指定归属，另一处删除；同名出现在同一卡片 → 合并为一条，note 保留信息量较大者
   - 特别关注常混淆项白名单：**充电器 / 充电宝 / 数据线 / 转换头** 各是独立物品，不视为重复；**充电线 / 数据线** 视为同名合并；**便携水壶 / 保温壶 / 水杯** 视为同名合并（保留最具体的那个）

7. **入境政策时效性水印**

   - 所有 info-card 中来自 §7 海关申报表 / §12 无人机法规 / regional-kits 的条目，末尾追加水印："信息截至 2026 年上半年，出发前请至驻外使领馆官网核查"
   - 临走前待办自动加入一条："T-30 天：复核签证 / 海关新政（可能有变化）"
