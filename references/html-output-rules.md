# HTML 输出详细规范

> 本文件从 SKILL.md 外置，包含页面结构、紧急联系区块、临走前待办、交互功能和 CSS 设计规范的完整细节。
>
> **读取时机**：进入 HTML 渲染阶段时加载。

## 紧急联系/提醒区块（info-card）

**样式规范**：白色卡片 + 22px 圆角 + 1px #EAEAEA 边框 + 0 2px 8px 阴影，与分类卡片视觉层级一致但更轻量。标题行使用 emoji 图标 + 文字（16px, 800 字重），下方用分隔线逐行排列内容。

**标题动态逻辑**（根据区块内容自动选择标题）：

| 内容组合 | 标题 | 图标 |
|---------|------|------|
| 仅紧急电话/医院地址 | 紧急联系信息 | 📞 |
| 仅特殊提醒（高反/安全/风俗等） | 出行提醒 | ⚠️ |
| 紧急电话 + 特殊提醒 | 紧急联系与出行提醒 | 📞 |

**内容行样式**：左侧 label（14px, 600 字重），右侧 value（14px, 700 字重，电话/紧急号码用红色 #EF4444），行间距 1px #EAEAEA 分隔线。

## 临走前待办区块（pre-departure）

**位置**：按天穿搭建议与紧急联系/提醒区之间，独立卡片样式。

**设计原则**：仅使用已采集字段（行程天数、交通方式、国内/国际）驱动，不额外追问用户。

**动态生成规则**：

| 条件 | 待办项 |
|------|--------|
| 行程天数 ≥ 5 | 关总水阀、关地暖/空调、清理冰箱、倒垃圾、浇花/委托照看 |
| 飞机出行 | 液体分装≤100ml/件 |
| 国际行程 | 护照签证最终确认、手机国际漫游开通、外币现金兑换 |
| 国内自驾（开自己的车） | 车况检查、ETC/加油卡、驾驶证/行驶证、车载充电器 |
| 国际自驾/租车 | 驾驶证+翻译件/国际驾照认证、租车确认单打印、车载充电器（租车公司可租） |
| 亲子游 | 儿童常用药确认、儿童座椅（自驾时） |

**样式规范**：与分类卡片视觉一致，标题使用 emoji + "临走前待办"，每个待办项为可勾选的 checkbox 行，支持自定义添加。

**交互**：与分类清单一致，支持勾选、删除、自定义添加、进度持久化。

> ⚠ **`add-item-row` 禁止硬编码**：HTML 模板中 `.pre-departure` 内不得出现 `.add-item-row` 元素。添加行由 JS `setupAddItemRows()` 统一动态注入到 `.pre-departure-items` 末尾。硬编码会导致 JS 容器检测失效、出现重复输入框。

## 交互功能

1. **勾选**
   - 点击物品名称或复选框切换状态
   - 勾选后文字加删除线 + 变灰
   - 进度条实时更新（已勾选/总数）

2. **删除（所有 item 通用）**
   - 鼠标 hover 任意物品（默认 + 自定义）右侧显示 × 按钮，点击即可删除
   - 删除状态持久化到 `localStorage[TRIP_KEY + '_deleted']`，刷新后保持
   - 删除时同时清除该 item 的勾选状态
   - 「重置进度」按钮清空 _deleted 并 reload，恢复全部默认项

3. **删除整个分类卡片**
   - 鼠标 hover `.category-header` 时右侧显示 × 按钮，点击弹二次确认（含分类名），确认后整卡从页面消失
   - 删除状态持久化到 `localStorage[TRIP_KEY + '_deleted_cats']`（存分类稳定 key = header 纯文本去 emoji/空白），刷新后仍不出现
   - 卡内所有 item 的勾选状态同步从 `TRIP_KEY` 主表清除，避免"看不见但仍占进度"的幽灵条目
   - 分类内的自定义 item 记录（`_custom`）**不**在此步逐条清理——因为 category-card 的 `data-cat` 索引在下次加载时会因缺卡而错位；统一由「重置进度」清 `_custom` 恢复
   - 打印 / 图片导出时 `.cat-delete-btn` 强制隐藏（`@media print` + `doExportAsImage` 双通道），不会印到纸/图上
   - 「重置进度」按钮清空 `_deleted_cats` 并 reload，恢复全部默认分类

4. **筛选**
   - 操作栏左侧提供「全部」「已打包」「待打包」三个筛选按钮，默认选中「全部」
   - 每个按钮后显示对应数量（如「已打包 5」「待打包 7」），数量实时更新
   - 切换筛选时，不符合条件的物品隐藏，整个分类卡片在无可见物品时也隐藏
   - 勾选/取消勾选后自动触发重新筛选（如筛选「待打包」时勾选一项，该项立即消失）

5. **本地存储**
   - 使用 localStorage 保存勾选状态、自定义物品、已删除的默认 item、已删除的分类卡片
   - 页面刷新后恢复进度
   - 存储 key 使用行程标识的 hash（目的地+日期），分别用 `TRIP_KEY` / `_custom` / `_deleted` / `_deleted_cats` 四个后缀

6. **打印**
   - 打印样式隐藏操作栏和进度条
   - 保留勾选状态（已勾选的显示 ✓）
   - A4 纸优化排版

7. **分享**
   - 生成包含行程参数的 URL（base64 编码）
   - 打开分享链接时自动恢复清单状态

## CSS 设计规范

- 主色调：#FFE600（VanMoof 荧光黄，卡片头部、进度条、筛选按钮高亮）
- 背景：#F4F4F4（浅灰）
- 卡片：#FFFFFF 白色，22px 大圆角，1px #EAEAEA 边框
- 文字：#0F0F0F（纯黑），次要文字 #7E7E7E
- 标签：统一灰色 #F0F0F0 背景 + #555 文字，无高亮变体
- 字体：'Plus Jakarta Sans', 'SF Pro Display', -apple-system, 'Noto Sans SC', 'PingFang SC', sans-serif
- 卡片头部：荧光黄 #FFE600 背景 + 纯黑文字，800 字重
- 勾选框：18×18px，accent-color 跟随主色
- 备注标签：胶囊形 #F4F4F4 背景
- **备注长度硬限**：`.note` 内文字不得超过 12 个中文字符（约 24 英文字符）。CSS 已设置 `max-width: 45%` + `text-overflow: ellipsis` 作为兜底，但 AI 生成 body 时应主动控制——超出时把关键信息移入 label（如"往返机票行程单（打印+电子版）"），note 只保留最短辅助说明
- 筛选按钮：胶囊形按钮组，active 态荧光黄
- 阴影：0 2px 8px rgba(0,0,0,0.04)
- 信息卡片（info-card）：白色背景 + 22px 圆角 + 1px #EAEAEA 边框，标题行 emoji + 16px/800 字重，内容行 label 14px/600 + value 14px/700（紧急号码红色 #EF4444），行间 1px #EAEAEA 分隔线

## 脚本结束标签强制规则

`<script>` 块的结束标签必须是**未转义的** `</script>`。严禁使用 `<\/script>` 作为脚本块的结束标签——浏览器 HTML 解析器只认 `</script>` 来关闭脚本块，`<\/script>` 会被当作普通 JS 内容，导致整个 `<script>` 块永不关闭、所有 JavaScript 代码不执行。仅在 JS 字符串字面量内部出现 `</script>` 时才需要转义为 `<\/script>`（防止提前终止脚本块），脚本块本身的结束标签永远不转义。

## 信息不完整模式（approximate 与 approximate_days）

`approximate=true`（日期未定）与 `approximate_days=true`（天数未定）共用同一套"降级渲染"逻辑，只在头部横幅文案上区分：

- **头部横幅**：日期未定 → `.approx-notice` "📅 出发日未定，按 {目的地} {月份} 气候常态估算，建议出行前 3 天回看本清单"；天数未定 → `.approx-days-notice` "📅 行程天数未定，按 7 天估算衣物件数，请根据实际行程加减"。两者同时命中时两条横幅都渲染。
- **按天穿搭**：任一标志为 true 都禁用 Day 1~Day N 逐日列表，改用「前半程穿搭」+「后半程穿搭」两段式，每段含温度区间 + 洋葱式分层建议。
- **行程 tag**：日期未定 → "{月份}（待定）"；天数未定 → "约 7 天"。
- **临走待办追加**：日期未定 → "出发前 3 天回看本清单，核对实时天气并补充/移除衣物"；天数未定 → "出发前一周确认实际行程天数，按比例调整衣物/洗漱用品数量"。
- **衣物件数提示**：仅天数未定时，数量敏感条目（衣物 / 洗漱）后追加"（按 7 天估算）"小字；证件、电子设备等非数量敏感的不加。
- **多城市**：天数未定时自动豁免各城市天数划分追问，按"快速版"整体温度范围渲染。

## 禁用减法建议模块

**不渲染** `.subtraction-section`「以下物品你不需要带」区块。主清单已按天气/出行类型/交通方式过滤过，再列一遍砍件信息属于冗余且打断用户"看清单 → 勾选"主流程。天气过滤、用户已声明的装备、酒店常识提示等场景，全部通过"主清单条目本身的取舍"或单条 note 解决，不再单列模块。

## JS 函数必须严格复制模板实现

`exportAsImage`、`doExportAsImage`、`copyAsText` 三个函数必须**原样复制** `assets/templates/html-template.html` 中的实现，**禁止简化或重写**——因为 html2canvas 渲染管线对 DOM 结构、class 名称和 CSS 选择器高度敏感，任何层级调整或简化都可能导致导出图片空白/样式丢失。关键要求：

1. `exportAsImage` 动态加载 html2canvas（CDN 不可用时降级 copyAsText）
2. `doExportAsImage` 使用 375px 移动端 wrapper + 自定义 checkbox 替代原生元素（html2canvas 无法渲染原生 checkbox）+ `canvas.toBlob` 导出 PNG + 中文文件名
3. `copyAsText` 完整提取 header/tags/分类清单/每日穿搭/紧急联系信息 + clipboard API 降级 execCommand

## 导出文件名 sanitize 规则

`exportAsImage` 中 `a.download` 从 h1 动态读取，但 h1 内容可能含浏览器不允许的字符（`/` `:` `?` `*` `<` `>` `|` `\` `"`），会导致下载失败或文件名被截断。JS 中必须做一次 sanitize：

```js
var raw = (document.querySelector('h1') || {}).textContent || '行李清单';
var safeName = raw.replace(/[\\/:*?"<>|]/g, '-').replace(/\s+/g, '_').slice(0, 80);
a.download = safeName + '.png';
```

中文/英文/数字/破折号/下划线全部保留，仅替换非法字符为 `-`。80 字符截断防止极端长标题。同规则应用于 `copyAsText` 的下载文件名（如未来加入 .txt 导出）。

## 备注 hover 高度上限

`.note:hover` 自然换行展开时，若备注 > 24 中文字，容器高度可能撑破相邻卡片布局。CSS 强制：

```css
.note:hover {
  max-height: 96px;
  overflow-y: auto;
}
```

且 AI 生成 body 时应主动把备注控制在 ≤ 24 中文字（原 12 字规则的上限放宽给"确实需要展开看的说明"，仍避免长篇大论）；> 24 字的内容拆一半移到 label 或做二次省略。

## 长清单导出分页警告

当 category-card 总数 ≥ 8 或 item 总数 ≥ 120 时：
- 生成阶段：在 body 末尾追加隐藏元素 `<div data-long-list="true">`，供 JS 探测
- `doExportAsImage` 前置检测：wrapper.scrollHeight > 16384（部分浏览器 canvas 高度上限）时提示 `"清单过长（{N} 项），导出图片可能被截断。建议使用「打印」功能分页保存 PDF。"`，用户点确认继续导出，否则取消
- 不阻塞导出，只提示

## 临走待办类型白名单

`.pre-departure` 内所有待办项必须符合"离家前动作 + 家门口场景"，通过白名单过滤：

- ✅ 允许：关水阀 / 关地暖 / 清冰箱 / 倒垃圾 / 浇花 / 委托照看宠物 / 液体分装 / 护照最终确认 / 手机漫游开通 / 外币兑换 / 车况检查 / ETC 检查 / 儿童座椅确认 / 儿童常用药确认
- ❌ 禁止：机票预订 / 酒店预订 / 打车预约 / 大巴订票 / 行程规划 / 目的地攻略——这些属于"行程规划"域，不归本 skill

生成时逐条过白名单，命中禁词直接过滤掉；无法明确判断的通用建议（如"整理证件"）默认允许，但优先归入证件卡的可勾选项而非临走待办。
